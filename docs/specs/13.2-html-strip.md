# 13.2 — Limpieza de HTML en títulos y extensión del overlay

Sub-spec del 13. Resuelve un caso concreto detectado tras ejecutar el spec 13:

> ORCID guarda algunos títulos con marcado HTML inline para preservar matices tipográficos científicos. En el cache aparecen como texto literal:
> 
> `<span style="font-variant: small-caps">l</span>-Carnitine and Alpha-Lipoic Acid...`

Verificado: **solo 1 paper afectado de 276** (el de L-Carnitine en *Nutrients*). Pero es un fix preventivo: si en el futuro entran más papers con HTML, el strip los cubre automáticamente.

## Dos cambios

### Cambio 1 — Strip de HTML en el script

Función defensiva en `scripts/fetch-orcid.ts` que elimina tags HTML del título y journal antes de guardarlos en el cache.

```typescript
function stripHtml(text: string): string {
  if (!text) return '';
  return text
    .replace(/<[^>]+>/g, '')           // tags HTML
    .replace(/&amp;/g, '&')            // entidades comunes
    .replace(/&lt;/g, '<')
    .replace(/&gt;/g, '>')
    .replace(/&quot;/g, '"')
    .replace(/&#39;/g, "'")
    .replace(/&nbsp;/g, ' ')
    .replace(/\s+/g, ' ')              // colapsa espacios
    .trim();
}
```

Aplicarla en `mapOrcidWork` (o función equivalente):

```typescript
title: stripHtml(work.title?.title?.value ?? ''),
journal: stripHtml(work['journal-title']?.value ?? ''),
```

**No usar librería externa** (`sanitize-html`, `dompurify`, etc.). Para 1 caso aislado no compensa la dependencia.

### Cambio 2 — Extender overlay para permitir sobreescribir `title`

Editar `lib/data/publications-merged.ts`. Cambiar:

```typescript
type OverlayEntry = Partial<
  Pick<Publication, 'isFeatured' | 'topics' | 'quartile' | 'impactFactor' | 'citations'>
>;
```

Por:

```typescript
type OverlayEntry = Partial<
  Pick<Publication, 'isFeatured' | 'topics' | 'quartile' | 'impactFactor' | 'citations' | 'title'>
>;
```

Solo añade `'title'` al `Pick`. La función `applyOverlay()` ya hace `{ ...p, ...o, ... }` así que automáticamente respetará el title si está definido en el overlay.

### Cambio 3 — Curar L-Carnitine en el overlay

Editar `lib/data/publications-overlay.json` y añadir entrada:

```json
{
  "__comment": "Overlay manual sobre datos de ORCID. Claves = DOI (en minúsculas). Valores = campos a sobreescribir (isFeatured, topics, quartile, impactFactor, citations, title). Editar a mano. Specs 13 y 13.2.",
  "10.3390/nu17132227": {
    "title": "ʟ-Carnitine and Alpha-Lipoic Acid Fail to Improve Anaerobic and Aerobic Performance in Trained Cyclists Despite a Reduction in Blood Lactate Concentration"
  }
}
```

**Detalle crítico**: el carácter `ʟ` es Unicode **U+029F** (LATIN LETTER SMALL CAPITAL L), no la letra `l` minúscula normal ni HTML. Es la nomenclatura química correcta para el isómero levógiro y se renderiza con versalita real en cualquier navegador sin necesidad de marcado.

**Otro detalle**: la clave DOI va en minúsculas (`10.3390/nu17132227`, no `10.3390/NU17132227`), porque `applyOverlay()` ya hace `p.doi.toLowerCase()` para el lookup. Si tu cache tiene el DOI en otra capitalización, igualmente matchea.

## Por qué no Opción B (renderizar HTML sanitizado)

Considerada y descartada. Para **1 paper en 276**:

- Añadir `isomorphic-dompurify` o `sanitize-html`: ~30 KB
- Cambiar `<h3>{title}</h3>` por `<h3 dangerouslySetInnerHTML={...} />`: introduce vector XSS si la sanitización falla
- Configurar allowlist de tags: superficie de error nueva
- Mantenimiento: cada update de la librería de sanitización requiere revisión

A cambio de poder renderizar `<span style="font-variant: small-caps">l</span>` fielmente. El carácter Unicode `ʟ` resuelve el caso real sin nada de eso.

**Si en el futuro aparecen papers con tipografía científica compleja** (subíndices reales `VO₂max`, fórmulas químicas, nomenclatura biológica con itálicas), reabriremos como Spec 13.3 con análisis específico. Por ahora, overhead innecesario.

## Patrón para casos similares futuros

Si en el próximo `pnpm fetch:orcid` aparece otro paper con tipografía especial:

1. El strip lo deja como texto plano (sin pérdida de info legible).
2. Si la tipografía importa (ej. nomenclatura química), añadir entrada al overlay con el título usando Unicode apropiado.
3. Caracteres útiles:
   - Versalitas: `ʟ` (U+029F), `ᴅ` (U+1D05)
   - Subíndices: `₀` `₁` `₂` `₃` `₄` `₅` `₆` `₇` `₈` `₉`
   - Superíndices: `⁰` `¹` `²` `³` `⁴` `⁵` `⁶` `⁷` `⁸` `⁹`
   - Letras griegas: `α β γ δ ε ζ η θ λ μ π σ ω`
