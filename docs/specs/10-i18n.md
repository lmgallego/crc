# 10 — i18n completa (ES / EN)

Spec dedicado a asegurar que todo el contenido funcione perfectamente en ambos idiomas.

## Reglas generales

1. **ES es default**. URL: `/`, `/filosofia`, etc.
2. **EN es prefijado**. URL: `/en`, `/en/philosophy`, etc.
3. **Las URLs SÍ se traducen** en next-intl con `localePrefix: 'as-needed'` y un mapping de `pathnames`.

## Pathnames traducidos

`i18n/routing.ts`:

```typescript
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['es', 'en'],
  defaultLocale: 'es',
  localePrefix: 'as-needed',
  pathnames: {
    '/': '/',
    '/filosofia': {
      es: '/filosofia',
      en: '/philosophy',
    },
    '/ciclista-3-0': {
      es: '/ciclista-3-0',
      en: '/cyclist-3-0',
    },
    '/servicios': {
      es: '/servicios',
      en: '/services',
    },
    '/servicios/[slug]': {
      es: '/servicios/[slug]',
      en: '/services/[slug]',
    },
    '/equipo': {
      es: '/equipo',
      en: '/team',
    },
    '/equipo/[slug]': {
      es: '/equipo/[slug]',
      en: '/team/[slug]',
    },
    '/publicaciones': {
      es: '/publicaciones',
      en: '/publications',
    },
    '/blog': '/blog',
    '/blog/[slug]': '/blog/[slug]',
    '/contacto': {
      es: '/contacto',
      en: '/contact',
    },
  },
});
```

## Slugs de servicios localizados

| ES | EN |
|---|---|
| `/servicios/investigacion` | `/services/research` |
| `/servicios/coaching` | `/services/coaching` |
| `/servicios/training-camps` | `/services/training-camps` |
| `/servicios/formacion` | `/services/education` |

Configuración:

```typescript
'/servicios/investigacion': { es: '/servicios/investigacion', en: '/services/research' },
'/servicios/coaching': { es: '/servicios/coaching', en: '/services/coaching' },
'/servicios/training-camps': { es: '/servicios/training-camps', en: '/services/training-camps' },
'/servicios/formacion': { es: '/servicios/formacion', en: '/services/education' },
```

Los slugs internos del equipo (`mikel-zabala`, `manuel-mateo-march`...) NO se traducen, son los mismos en ambos idiomas.

## Qué se traduce y qué NO

### Sí se traduce
- Toda la UI (nav, footer, botones, labels).
- El manifiesto del paradigma 1.0/2.0/3.0 entero.
- Las descripciones de servicios.
- Los textos editoriales de la home.
- El blog (cada artículo se escribe manualmente en cada idioma).
- Las descripciones cortas y biografías de los miembros.
- Las roles visibles ("Director", "Senior researcher").

### NO se traduce
- Los nombres de los miembros (Mikel Zabala es Mikel Zabala en EN).
- Los títulos de las publicaciones científicas.
- Los nombres de revistas (`Journal of Science and Cycling`).
- Los grupos de investigación (`HUM1063`).
- Las certificaciones (`NSCA-CSCS`, `ISAK`).
- Los nombres propios de organizaciones (`Movistar Team`, `RFEC`).
- Las ubicaciones (Granada, Barcelona, Almería) — quedan en español incluso en EN.

## Validación

Antes de declarar i18n completa:

1. Comprobar que `/` en castellano no muestra ningún string en inglés.
2. Comprobar que `/en` en inglés no muestra ningún string en español.
3. Cambiar de idioma en cualquier página → debe llevarte a la equivalente en el otro idioma sin romper la URL.
4. SEO: cada página tiene `<html lang="es">` o `<html lang="en">` correcto.
5. Cada página tiene `<link rel="alternate" hreflang="es" href="..." />` y `hreflang="en"` para SEO.

## Metadata por página

`app/[locale]/layout.tsx`:

```typescript
import { getTranslations } from 'next-intl/server';

export async function generateMetadata({ params }) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: 'metadata.home' });
  return {
    title: t('title'),
    description: t('description'),
    alternates: {
      canonical: locale === 'es' ? '/' : `/en`,
      languages: {
        es: '/',
        en: '/en',
      },
    },
  };
}
```

Cada página debe definir su `generateMetadata` con título y descripción específicos en ambos idiomas.
